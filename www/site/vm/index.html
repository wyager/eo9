Things to try

  • hello --name web --excited true run a real Eo9 program: typed arguments in, a typed outcome (greeted) out — no argv, no exit-code guessing.
  • rng --count 3  then  entropy.seeded --seed 43 $ rng --count 3 virtualization by substitution: the first uses your browser's real randomness; the second composes a seeded RNG over the same unmodified program. Run it twice — identical output. The fused composition is compiled right here in the page.
  • only eo9:text/text,eo9:time/time $ hello --name boxed --excited true capability lockdown that passes: the allow-list grants exactly what hello needs, so it runs.
  • only eo9:text/text $ hello --name boxed --excited true capability lockdown that refuses: drop the clock from the allow-list and the program is rejected before it runs — the import it needs simply isn't there.
  • time.frozen --now-seconds 0 --monotonic-ns 0 $ hello --name frozen --excited true a virtualized clock: the program asks the OS for the time and gets exactly the instant you composed in — its timestamp reads 0.000000000, every run, in any browser.
  • ls /  and  cat /welcome.txt /docs/about.txt real coreutils (also Eo9 components) against an in-memory filesystem provided by the OS — the same programs run unchanged on native Eo9 and on the bare-metal kernel.

exit ends the session; reload the page for a fresh one.

What this is actually doing

This is the full Eo9 operating system — the real runtime, the capability algebra, the compiler, and the unmodified eosh shell — compiled to WebAssembly and running entirely inside this page. Programs are resolved from /bin, spawned through the same typed eo9:exec surface the native CLI and the bare-metal kernel use, and compositions you type at the prompt are fused by the real component algebra and compiled in the page.

The one difference from the native and bare-metal versions: compiled programs here run on a portable bytecode interpreter (Pulley) instead of as native machine code, because there is no wasm→wasm code generator to hand the browser — so it is slower. On real hardware, Eo9 compiles programs and compositions to native code on the machine itself. Also not here: persistence (the filesystem is in-memory and resets on reload) and networking. Genuinely blocking operations (the prompt's read-line, time.sleep) suspend the whole VM with JavaScript Promise Integration, so this page needs a JSPI-capable browser — current Chrome or Edge.
