What this is

The buttons above drive a single ~1 MB WebAssembly module: the same wasmtime the native Eo9 runtime and the bare-metal kernel embed, compiled for wasm32, with its Pulley interpreter executing pre-compiled images of unmodified Eo9 components. The page's JavaScript only provides this output box; no Eo9 semantics are re-implemented in JS.

  • hello + add — the kernel's seed component (the first thing Eo9 ever ran on bare metal), executed by Pulley inside your browser.
  • fuel metering — the same call with wasmtime's fuel accounting on, the mechanism Eo9 uses to meter CPU.
  • entropy.seeded — an unmodified Eo9 provider stub using the Component Model async ABI: its configure and get-u64 are async-lifted calls, executed here without fibers (wasm32 has none) via Eo9's fiberless execution path. Same seed, same sequence as native Eo9 and the bare-metal kernel.

What is not here yet: the shell (eosh), the capability algebra, the store, and programs that block on real I/O. Those are the next milestones for this page; the /try page remains the polished demo in the meantime.
